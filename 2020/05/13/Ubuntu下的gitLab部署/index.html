Ubuntu下的gitLab部署

Gitlab部署

安装依赖：

$ sudo apt-get update
$ sudo apt-get install -y curl openssh-server ca-certificates

# 安装postfix，选择Internet Site
$ sudo apt-get install -y postfix
$ curl https://packages.gitlab.com/install/repositories/gitlab/gitlab-ee/script.deb.sh | sudo bash
$ sudo apt-get install gitlab-ee

Nginx反代理

将/etc/gitlab/gitlab.rb中的729行、731行改为

gitlab_workhorse['listen_network'] = "tcp"
gitlab_workhorse['listen_addr'] = "127.0.0.1:[port]"

其中[port]为自定义端口，需要与nginx的反代理端口一致。

运行sudo gitlab-ctl reconfigure更改生效。

在nginx配置文件中，增加gitlab服务:

$ sudo vim /etc/nginx/conf.d/gitlab.conf

server {
    listen       80;
    server_name  lab.chensh236.com;

    location / {
        root  html;
        index index.html index.htm;
        proxy_pass http://127.0.0.1:[port];
    }
}

运行sudo systemctl restart nginx生效。

数据迁移

$ mkdir *
$ chmod -R git:root *

将/etc/gitlab/gitlab.rb中的484行改为新路径。

邮件服务

在gitlab.rb中进行设置（620行）

gitlab_rails['smtp_enable']=true
gitlab_rails['smtp_address']="smtp.exmail.qq.com"
gitlab_rails['smtp_port']=465
gitlab_rails['smtp_user_name']="name" # 只填写账号
gitlab_rails['smtp_password']="pass"
gitlab_rails['smtp_domain']="doamin.com"
gitlab_rails['smtp_authentication']="login"
gitlab_rails['smtp_enable_starttls_auto']=true
gitlab_rails['smtp_tls']=true
gitlab_rails['gitlab_email_from']="name@domain.com"

执行sudo gitlab_ctl reconfigure即可。

关于 Permission denied

在Windows的凭据管理器中删除重新添加即可。
